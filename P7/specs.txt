CSE 325 - Project 7

Project Overview
This project extends your work from Project 6 by adding a virtual memory layer between the processor
and physical RAM. You will design and implement a C/C++ program that simulates a virtual memory
system with per-process page tables, a shared physical RAM, and a disk swap area. The simulation
handles page faults, page replacement using the FIFO (First-in-First-out) policy, dirty-bit write-back,
and memory protection faults. Please submit your work for grading via the D2L system.

Project Specifications
The system to be simulated consists of multiple concurrent processes (P0, P1, P2, etc), each with its
own virtual address space and page table, sharing a common physical RAM and a disk swap area. The
parameters of the system are fixed as follows:
Virtual address bits:

8 (4-bit VPN + 4-bit byte offset)

Number of Pages per process:

4, 8, or 16

Page size:

16 bytes (offsets 0x0–0xF)

Physical RAM frames:

8 (frames 0-7, shared across all processes)

Disk swap slots:

32 (slots 0-31)

Replacement policy:

FIFO (first-in, first-out)

Each page table entry contains four fields:
• P (present bit): 1 if the page is currently in RAM, 0 if it is on disk or unallocated.
• M (modified bit): 1 if the page has been written since it was loaded into RAM. A dirty page must
be written back to disk before its frame is reused.
• A (access bit): A=1 if the user can access the page. An attempt to access when A=0 raises a
protection fault; no state is modified.
• Frame: when P=1, the physical frame number holding the page;
• Disk Area: The area in the disk where the corresponding page is saved

5. Page replacement uses the FIFO policy. The program must maintain a queue of occupied frames in
the order in which they were loaded. When a page fault occurs and all frames are occupied, the frame
at the head of the queue (the oldest resident page) is evicted. If that frame’s dirty bit is set, its contents
are written back to the disk slot associated with the evicted page before the frame is reused. The evicted
process’s page table entry is updated: P is set to 0 and the Frame/Slot field is updated to the disk slot
number.
6. The program will include appropriate logic to handle the following exceptional cases:
• A protection fault occurs when a READ/WRITE targets a page whose A bit is 0. No page table
entry, dirty bit, RAM frame, or FIFO queue is modified. The output line records PROT FAULT and
PA=--.
• A page fault on a page that has never been accessed (Frame/Slot = “–”) allocates a new disk slot
as the backing store for the page.
• If the -input file cannot be opened, the program will print an appropriate error message and exit.
• If -ram or -disk option is not provided, RAM and Disk is initialized by all 0’s respectively.
